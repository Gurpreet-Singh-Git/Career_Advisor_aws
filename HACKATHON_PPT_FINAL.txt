Slide 6: ML Ranking Engine (Technical)
Title: Explainable ML Career Ranking

Algorithm Details:

Input Vector:
• Education level (categorical)
• Skills array (text → embeddings)
• Interests array (text → embeddings)
• Location (categorical)

Scoring Function:
Total_Score = 0.35 × Skill_Match + 0.25 × Education_Fit + 0.25 × Interest_Alignment + 0.15 × Regional_Demand

Skill_Match = Jaccard_Similarity(user_skills, career_skills)
Education_Fit = Hierarchy_Match(user_education, required_education)
Interest_Alignment = Cosine_Similarity(user_interests, career_interests)
Regional_Demand = Demand_Score(location, career)

Output: Top 3 careers with score breakdown

Slide 8: System Flow - Technical
Title: End-to-End Request Flow

Step-by-step technical flow:

1. User Input → CloudFront → S3 (React app loads)
2. User message → API Gateway → Lambda (intent_detection)
3. Intent = "personalized" → Show consent dialog
4. User submits form → API Gateway → Lambda (rank_careers)
5. Lambda loads careers from S3
6. ML engine computes scores (4 components)
7. Top 3 careers selected
8. For each career → Lambda calls Bedrock API
9. Bedrock generates explanation (Claude 3 Sonnet)
10. Response → API Gateway → CloudFront → User

Latency: <2 seconds end-to-end
Concurrent users: Auto-scales (Lambda)
